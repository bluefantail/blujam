The specificest game jam in Wellington

BluJm is a 48-hour game making competition with some very unique rules.

  • You must use pure Java and the ECS100 library to make your game
  • No external libraries are allowed
  • Your game must be presented primarily through the UI class in the ECS100 library

BluJm starts at 6pm on the Friday with the announcement of the theme, and runs until 6pm Sunday followed by voting and prize giving. Everyone from first-year university students to seasoned game makers are welcome to sign up and make some games.

Your game is only allowed to use the Java standard library, the ECS100 library, any code that we provide, and any pure Java that you write yourselves at or before BluJm. This means no LWJGL, JOGL, JNI, third-party game or physics engines etc. While you must use the window provided by the ECS100 UI class, you are allowed to modify the JFrame, although we discourage just removing all of its components.

There are no restrictions on the amount of preparation your team can make before the event, but we discourage you from building for a specific game. The theme is only announced at the start of the competition, and we want you to turn up with an open mind for potential games.

Beginners vs Veterans

Beginners

  • Practise creating code without a strict guideline
  • Learn more about game making and team development
  • Test your skills against seasoned programmers

Veterans

  • Practise making good games with basic tools
  • Share your knowledge of game making with like-minded people
  • Test your programming skills in a restricted domain
